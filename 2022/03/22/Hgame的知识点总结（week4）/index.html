Xml（可扩展标记语言）与html（超文本标记语言）比较类似，当xml的外部实体解析开启（允许了外部实体的加载）之后，就会存在xxe注入的问题

XXE漏洞全称XML External Entity Injection，即外部实体注入漏洞，XXE漏洞发生在应用程序解析XML输入时，没有禁用外部实体的加载，导致可加载恶意外部文件，造成文件读取、命令执行、内网端口扫描、攻击内网网站等危害

建议：禁用xml外部实体解析

Js语句当中try和catch一般都是成对出现

toUpperCase()的解读：在本例当中获取password对象的toUpperCase属性，然后把这个属性当作函数来调用，说简单一点就是将password的一整个字符串的全部小写字母转化为大写字母，如果它不是一个字符串（可能是一个数组对象）的时候就会报错

Js原型链污染

Zombie(僵尸)利用zombie包的漏洞打出一个rce(远程命令/代码执行漏洞)

参考了一下summ3r学长的博客：https://blog.summ3r.top/2021/08/26/Nodejs-Zoombie-Package-RCE-%E5%88%86%E6%9E%90/

Zombie是一个轻量级框架，用于在模拟环境中测试客户端的js代码，甚至都不要浏览器的支持（nb），此漏洞的zombie版本为6.14，截至2021/8/26，该6.14版本发布于2018年

具体漏洞分析可参考上述博客

最终的结果是使用node.js的模块运行js代码，代码运行的上下文就是 window 对象

关于vm逃逸

Vm并不是一个绝对安全的方案机制，不应该用它来执行不安全的代码

Vm逃逸的个人理解：逃脱出这个名为vm的模块，然后脱离出vm虚拟机的有限空间，进而影响到外界系统主机，典型的一点是某个病毒逃脱vm虚拟机，然后该病毒感染了主机，像极了逃出了牢笼的狮子

攻击者构造包含恶意 javascript 代码的页面，使用 Zoombie 访问后就有可能产生 vm 逃逸实现 RCE

转载自summ3r的博客的部分截图以及分析https://blog.summ3r.top/2021/08/26/Nodejs-Zoombie-Package-RCE-%E5%88%86%E6%9E%90/

使用proto访问到了属于父对象的内容，另一种意思即为逃脱了context的全局空间，以此使用了一些不属于context全局空间对象，造成了vm逃逸（估计原本的vm模块当中是没有这个对象的）（__proto__、constructor等都属于一个对象内置的方法）

即为借助了原型链（prototype、**proto **）访问到了vm沙盒以外的内容，实现了RCE

例如：子对象.proto=（等同于，不是赋值）父对象.prototype，由此访问到属于父对象的内容

我又参考了另一个博客https://www.cnblogs.com/l0nmar/p/13951739.html

链的理解：一种逻辑顺序

一些对象的内置方法：

Prototype（原型，也可以理解为一个对象的母体或是心脏）:prototype是一个类的属性，所有类对象在实例化（定义define）的时候将会拥有Prototype中的属性和方法

当调用一个对象的prototype的时候，会回显出这个对象的prototype（原型）中定义（插入）的属性块（零件/插件）（好难解释，大致理解一下……）

例如 var inst = new MyClass()此时inst拥有了MyClass类的prototype属性

Constructor:通过这个属性可以查看对象的构造函数

proto:一个对象的__proto__属性（方法），指向这个对象所在类的prototype

注：每一个对象都有两个特殊的属性prototype、proto

类在运行程序运行时是可以修改的

实例如下图，调用了prototype属性为类添加新的属性

通过修改原型链来继承其他类（js中prototype链的继承机制）

以及通过将一个类对象定义赋值到一个新的变量这两种方法来创建父对象和子对象（个人理解）

可以调用多次__proto__:其中var stud=new student();每多一个__proto__，辈分关系+1（超级加辈）
